# Конвертация метрики → «американская» пародийная система

Справочник по единицам отображения **MyHealthDashboard**. Данные в API и SQLite **всегда метрические**; пародийные единицы применяются только в UI (и на осях графиков Plotly).

---

## Принцип

| Слой | Единицы |
|------|---------|
| БД, REST API, формы ввода | **Метрика** (кг, см, км, ккал, °C, Вт, г) |
| Отображение при `units_system = american` | **Пародийные** (Jp, SoL, iCharge, …) |

Переключатель: **Настройки → Интерфейс** → «Система единиц» (`metric` | `american`).  
Сохранение: `GET/POST /api/user/profile` (поле `units_system`). Отдельного `/api/interface-settings` нет.

### Тренировки

| Колонка UI (metric) | american |
|---------------------|----------|
| Ккал пульсометр | iCharge пульсометр |
| Ккал часы | iCharge часы |
| Дистанция | SoL / torch |
| Скорость | SoL/h |
| Вес силовых | Jp / Camry |

Хелперы: `useUnits`, `chestStrapKcal()` в `frontend/src/utils/format.ts`.

### Код

| Слой | Файл |
|------|------|
| Константы и формулы | `frontend/src/utils/americanUnits.ts` |
| Зеркало (Python) | `backend/utils/american_units.py` |
| Хук UI | `frontend/src/hooks/useUnits.ts` |
| Оси Plotly | `frontend/src/utils/units.ts` (`convertArrayKmToSol`, …) |

## Сводная таблица

| Метрика (хранение) | Пародийная единица | Коэффициент / правило | Где в UI |
|--------------------|--------------------|------------------------|----------|
| **г** (еда, БЖУ) | гран (gr), бандл., меш. | 1 г = 15.4323584 гран; бандл = 1256.5 гран; меш = 9985.33 гран | Дневник питания, продукты |
| **кг** (тело, вес) | **Jp** (японец) | 1 Jp = 62.5 кг | Вес, состав тела, lean mass |
| **кг** ≥ 80 (тело/снаряд) | **Camry** | 1 Camry = 1500 кг | Тяжёлые веса, штанга, 1ПМ |
| **кг** (нагрузка силовая) | **Jp** | 1 Jp = 62.5 кг | Объём, тоннаж |
| **см** (рост) | **Tp** (трамп) | 1 Tp = 190.5 см | Профиль, рост |
| **см** &lt; 50 (мелкие длины) | **Dk** (дик) | 1 Dk = 13.5 см | Длина шага, мелкие размеры |
| **см** ≥ 50 | **Tp** | через `formatLength` | Длинные отрезки |
| **км** (дистанция) | **SoL** (статуя Свободы) | 1 SoL = 93 м | Бег, вело, бассейн, шаги |
| **м** &lt; 46.5 м (&lt; 0.5 SoL) | **torch** (факел) | 1 torch = 9.3 м (= SoL/10) | Короткие дистанции |
| **км/ч** (скорость) | **SoL/h** | `км/ч × 1000 / 93` | Кардио, карта вело |
| **мин/км** (темп бега) | **мин/SoL** | `мин/км × 0.093` (= × 93/1000) | Бег, тренды |
| **м** (высота) | **рашмор** (Rushmore) | 1 рашмор = 1745 м | Карта, графики вело |
| **ккал** | **iCharge** | 1 iCharge = 12.74 ккал | Питание, кардио, аналитика |
| **Вт** (мощность) | **iCharge/мин** | через ккал/мин ÷ 12.74 | Вело (оценка/датчик) |
| **сек** (длительность) | **FEP** / **SB** | FEP = 22 мин; SB = 30 с | Длительность (american) |
| **°C** | **°Rj** (Rankin Junior) | нелинейная формула | Температура на карте |
| **мл** | **syr** (шприцы) | `0.5 × (√мл + 2) × 1.8` | Объёмы (если используется) |
| **г/сут** (темп похудения) | гран/ч | граны в сутки ÷ 24 | Сушка/набор |

Обхваты тела (талия, бедро, …) в american-режиме остаются в **см** (не Tp/Dk).

## Формулы по группам

### Дистанция (SoL / torch)

```
SoL     = метры / 93
torch   = метры / 9.3
км → SoL = (км × 1000) / 93
```

**Формат строки** (`formatDistanceMeters`):

- если `SoL < 0.5` → показать **torch** (1 знак): `12.3 torch`
- иначе → **SoL** (2 знака): `5.38 SoL`

Ввод в формах кардио: по-прежнему **км**.

### Скорость

```
SoL/h = (км/ч × 1000) / 93
```

Пример: 25 км/ч ≈ **268.82 SoL/h**.

### Темп (бег)

```
мин/SoL = мин/км × (93 / 1000) = мин/км × 0.093
```

Пример: 5:00 мин/км (5.0) → **0.465 мин/SoL**.

Бассейн в графиках: сек/100 м переводится через эквивалент `мин/км = (сек/100м × 10) / 60`, затем в мин/SoL.

### Высота

```
рашморы = метры / 1745
```

(5725 футов ≈ 1745 м на одну «гору Рашмор».)

### Вес

```
Jp     = кг / 62.5        (если кг < 80 для отображения тела/снаряда)
Camry  = кг / 1500        (если кг ≥ 80)
```

Обратно:

```
кг = Jp × 62.5
кг = Camry × 1500
```

### Рост и мелкие длины

```
Tp = см / 190.5
Dk = см / 13.5
```

Длина шага в истории шагов: `step_length_m × 100` → см → **Dk**.

### Энергия и мощность

```
iCharge = ккал / 12.74

iCharge/мин = (Вт × 60 / 4184) / 12.74
```

### Еда (граммы)

```
граны   = г × 15.4323584
бандлы  = граны / 1256.5
мешки   = граны / 9985.33
```

Пороги UI:

- &lt; 100 г → граны  
- 100 г … 5000 г → бандлы  
- ≥ 5000 г → мешки  

### Температура (°Rj)

```
°Rj = 100 × sin((°C + 10) × π/180) + 50 × ln(°C + 20) + 20   (если °C + 20 > 0)
```

### Время

```
FEP = секунды / (22 × 60)
SB  = секунды / 30
```

| Шаг | 0.75 м | 75 см → **5.56 Dk** |
| Подъём | 350 м | **0.201 рашмора** |
| Обед | 650 ккал | **51.0 iCharge** |

## Использование в коде (TypeScript)

```typescript
import {
  formatDistanceKm,
  kmhToSolPerHour,
  paceMinPerKmToMinPerSol,
  cmToDick,
  kgToAmericanWeight,
  kcalToIcharge,
} from "../utils/americanUnits";

import { useUnits } from "../hooks/useUnits";

// В компоненте — предпочтительно хук (учитывает профиль):
const { formatDistance, formatSpeed, formatPace, formatBodyWeight, formatSmallLength } = useUnits();

formatDistance(10.5);           // "112.90 SoL" или "… torch"
formatSpeed(28);                // "301.08 SoL/h"
formatPace(5.0);                // "0.47 мин/SoL"
formatSmallLength(75);          // "5.56 Dk" (длина шага в см)
```

Графики Plotly:

```typescript
import { convertArrayKmToSol, convertArrayKmhToSol, DISTANCE_AXIS_AMERICAN } from "../utils/units";
```

## Где применяется во фронтенде

| Раздел | Форматтеры |
|--------|------------|
| Кардио (бег/вело/бассейн) | `formatDistance`, `formatSpeed`, `formatPace`, `formatEnergy` |
| `BikeGpsMap`, `BikeWorkoutCharts` | SoL, SoL/h, рашморы, °Rj |
| `HeartRateChart`, `CardioMetricChart` | ось дистанции SoL, hover |
| Шаги (`StepsHistoryView`) | `formatDistance`, ось SoL, `formatSmallLength` для шага |
| Тело / вес | `formatBodyWeight`, `formatBarbellWeight`, `formatHeight` |
| Питание | `formatFoodWeight`, `formatEnergy` |
| Силовые / 1ПМ | `formatBarbellWeight`, график `OneRmChart` (Jp/Camry) |
| Настройки вело | вес райдера в Jp при вводе (american) |

**Не переводится:** ввод дистанции (км), веса (кг), калорий в формах; обхваты — см; масса велосипеда в настройках — кг.

## Legacy

В `americanUnits.ts` / `american_units.py` оставлены **мили** (`kmToMiles`, `KM_TO_MILES`) — deprecated, в UI не используются.

Плавание: `kmhToLinksPerMin` (звенья цепи) — в UI скорость бассейна показывается как **SoL/h**, как и остальное кардио.
