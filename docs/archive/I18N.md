# Локализация Forma (i18n)

## Структура

- Общие словари: [`shared/i18n/locales/`](../shared/i18n/locales/)
  - `ru.json` — основной язык
  - `en.json` — fallback
- Движок: [`shared/i18n/index.ts`](../shared/i18n/index.ts) — функции `t(key, params?)`, `i18n.setLocale('ru' | 'en')`

## Подключение

| Клиент | Импорт | Provider |
|--------|--------|----------|
| Web / Desktop | `import { useT } from "@/i18n"` | `I18nProvider` в `frontend/src/main.tsx` |
| Mobile | `import { useT } from "../i18n"` | `I18nProvider` в `mobile/App.tsx` |

## Правила

1. **Не хардкодить** пользовательские строки в JSX — только ключи `section.feature`.
2. Параметры: `t("backup.exportTableStep", { current: 1, total: 10, name: "food_entries" })`.
3. Технические термины (Legacy API, merge, debug) **не показывать** в production UI — использовать ключи из `modes.*`, `backup.*`, `sync.*`.
4. Dev-only UI — за `developerToolsEnabled` (desktop) или `__DEV__` (mobile), подписи из `dev.*`.
5. Новый экран: добавить ключи в **оба** JSON (`ru` и `en`).

## Миграция

Постепенно переносите экраны на `useT()`. Приоритет: вход, настройки, синхронизация, dashboard, onboarding.
